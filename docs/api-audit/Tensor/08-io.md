# Tensor — 08. I/O

> **Superset-method rollout** (Tensor, category 08 of 8 — the last category).
> The document is split into **Analysis** (the evidence — inventory, C++↔Python
> mapping, findings, arg ordering) and a self-contained **Recommendation** that
> is the *normative spec for the next version of Cytnx*: the next major version's
> `Tensor` I/O surface should be implemented to match §R exactly. Every
> behavioral claim is verified against the installed `cytnx==1.1.0` wheel by
> `docs/api-audit/probes/Tensor_08_io.py` (all `[PASS]`, exit 0).
> **No C++ probe accompanies this category.** The two divergences here are a
> *broken pickle protocol* (T-IO2 — a pure pybind-registration gap, no
> `py::pickle`) and an *asymmetric / cross-class-inconsistent file extension*
> (T-IO4 — a shared-C++ fact surfacing identically through the binding); neither
> is a `Tensor_conti.py`-vs-raw *binding* divergence, so gate 4 is recorded as
> *"no binding-fidelity finding"*. The direct analog is
> [UniTensor cat 11](../UniTensor/11-io-display.md) — the broken pickle is the
> same defect (UT-IO2), but the `_Load` **name** heap over-read (UT-IO5) has **no
> analog** here: a dense `Tensor` carries no name field, so its `Save`/`Load`
> round-trip is deterministically clean (T-IO3, the positive contrast).

**Category scope.** The members that serialize a Tensor to/from disk: the
structured `.cytn` path (`Save` member, `Load` static constructor) and the raw
headerless-binary path (`Tofile` member, `Fromfile` static constructor), plus the
half-present **pickle protocol** (`__getstate__` inherited from `object`;
`__setstate__` absent). The display side (`Tensor.__repr__`, `print()`) is
[cat 03](03-element-storage-access.md); `Init`/the constructor are
[cat 01](01-construction-init.md). C++ header:
`cytnx_src/include/Tensor.hpp:328` (`Save`), `:342` (`Tofile`), `:362` (`Load`),
`:388` (`Fromfile`); bodies `cytnx_src/src/Tensor.cpp:450-467` (`Save`),
`:470-488` (`_Save`, the on-disk layout — **no name field**), `:497-507`/`:509-533`
(`Load`/`_Load`, **no name read**), `:426-447` (`Tofile`), `:490-496`
(`Fromfile` → `Storage::Fromfile`). Python bindings:
`cytnx_src/pybind/tensor_py.cpp:286-288` (`Save`), `:289-290` (`Load`),
`:292-294` (`Tofile`), `:295-299` (`Fromfile`).

## A1. Current API (cytnx 1.1.0)

**Broken pickle (behavioral).** Despite `__getstate__` existing, `pickle.dumps(t)`
**raises** `TypeError: cannot pickle 'cytnx.cytnx.Tensor' object` — pybind11
registered **no** `py::pickle(...)` support, so the object is unpicklable
regardless of the inherited getstate. Probe: *"`pickle.dumps(t)` RAISES TypeError
… pickle is a broken stub (same defect as UniTensor UT-IO2)"*. `copy.deepcopy(t)`
**also** raises `TypeError` — but for an unrelated reason (its `__deepcopy__` is
bound to `clone` with no `memo` parameter, **cat 07**), so it never even reaches
the pickle fallback. Probe: *"`copy.deepcopy(t)` also RAISES TypeError …
(cross-ref cat 07)"*.

## A4. Argument ordering — positional & keyword

These are serializers, not constructors of rich metadata: each takes a path plus,
for the raw reader, the type/count needed to interpret headerless bytes.

| API | positional-required (in order) | operation parameters (keyword-capable) |
|---|---|---|
| `Save` (→`save`) | `fname` | *(none)* |
| `Load` (→`load`, static) | `fname` | *(none)* |
| `Tofile` (→`tofile`) | `fname` | *(none)* |
| `Fromfile` (→`fromfile`, static) | `fname`, `dtype` | `count=-1` |

- **Canonical positional rule (§R.0):** the path primary operand first, then the
  minimum metadata needed to interpret raw bytes (`dtype`), then optional
  operation params (`count`). Matches the live order; no reordering needed.
- **Naming:** `fname`/`dtype`/`count` are self-describing and correctly named
  (`py::arg` present on all four bindings, `py:286-299`). The only
  ordering-adjacent change is dropping `Save`'s deprecated no-extension
  auto-append (T-IO4) so `fname` is always an explicit path.

# R. Recommendation — normative spec for the next version

*This section is self-contained: it fully specifies the next-version `Tensor` I/O
surface. Implement Cytnx to match it. Findings above are the rationale; they are
not needed to implement §R.*

## R.0 Normative conventions (apply to every API in this category)

- **N-casing (SciPostPhysCodeb.53) — members lowercase (T-IO1).** `Save`→`save`,
  `Load`→`load`, `Tofile`→`tofile`, `Fromfile`→`fromfile`; all four are `Tensor`
  members (a static constructor is still a member), so they lowercase — the same
  decision as UniTensor UT-IO1. (Contrast the Capitalized *free* functions
  `Contract`/`Svd`, kept.)
- **A protocol is implemented fully or not at all (T-IO2).** Ship pickle only
  with **both** `__getstate__` and `__setstate__` (over the `.cytn` binary
  format); never a lone inherited `__getstate__` with no restore hook.
- **Structured serialization round-trips exactly (T-IO3).** `save`→`load` must
  preserve every stored field — values, shape, dtype, contiguity. A dense Tensor
  has no name, so there is no name over-read to fix here (the UT-IO5 memory bug is
  UniTensor-only) — but the guarantee must be documented.
- **Extension handling is symmetric, explicit, and unified (T-IO4).**
  `save`/`load` take a full path with extension; no one-sided auto-append. Use a
  **single** canonical extension shared with UniTensor (resolve the `.cytn` vs
  `.cytnx` split).
- **Raw I/O is metadata-free by design; document the hazard (T-IO5).**
  `tofile`/`fromfile` are the headerless raw path; `fromfile` returns 1-D and
  needs `dtype` (+`count`). This is deliberate (numpy-`fromfile` parity) — keep
  it, but steer users to `save`/`load` for shape/dtype fidelity.
- **Binding fidelity: none.** `save`/`load`/`tofile`/`fromfile` are direct pybind
  bindings over the C++ methods — no `Tensor_conti.py` wrapper, no leaked `c*`
  binding. Gate 4 (raw-C++ probe) is skipped: *no binding-fidelity finding*.

## R.1 Recommended API (exact signatures + behavior contract)

```python
class Tensor:
    # --- structured serialization (members → lowercase; the primary I/O) ---
    def save(self, fname: str) -> None: ...                       # write .cytn binary (was Save)
    @staticmethod
    def load(fname: str) -> "Tensor": ...                         # read .cytn binary (was Load)

    # --- raw headerless binary (numpy-fromfile-style interop) ---
    def tofile(self, fname: str) -> None: ...                     # write raw element bytes (was Tofile)
    @staticmethod
    def fromfile(fname: str, dtype: int, count: int = -1) -> "Tensor": ...  # flat rank-1 (was Fromfile)

    # --- pickle: implement BOTH over the save/load format, or remove entirely ---
    def __getstate__(self) -> bytes: ...                          # serialize (was inherited stub)
    def __setstate__(self, state: bytes) -> None: ...             # restore (was ABSENT)
```

`save`/`load`/`tofile`/`fromfile` are the renamed `Save`/`Load`/`Tofile`/
`Fromfile` (lowercase, `DeprecationWarning` aliases for one release). Pickle is
either implemented fully (both hooks, over the `.cytn` binary format) or the
misleading inherited `__getstate__` is removed. The auto-append is dropped and the
extension unified with UniTensor.

| `save` (was `Save`) | **rename** (T-IO1/IO3) | Member serializer: write this tensor to `fname` (explicit extension) as the structured binary format, preserving values, shape, dtype and contiguity — `load` reconstructs a value-equal tensor. *Migration:* `Save` kept as a `DeprecationWarning` alias for one minor release, then deleted. (No name over-read on the Tensor path — UT-IO5 is UniTensor-only.) |
| `load` (was `Load`) | **rename** (T-IO1/IO3/IO4) | Static deserializer: `Tensor.load(fname)` reads a structured file into a new Tensor, value-equal to the saved one. *Migration:* `Load` kept as a `DeprecationWarning` alias for one release. Require the full path (no asymmetric auto-append) and a unified extension (T-IO4). |
| `tofile` (was `Tofile`) | **rename** (T-IO1/IO5) | Member raw dump: write the tensor's element bytes headerless (row-major, contiguity forced) for numpy-style interop — no shape/dtype stored. *Migration:* `Tofile` kept as a `DeprecationWarning` alias for one release. Document that shape/dtype are NOT preserved (use `save` for that). |
| `fromfile` (was `Fromfile`) | **rename** (T-IO1/IO5) | Static raw reader: `Tensor.fromfile(fname, dtype, count=-1)` reads `count` elements of `dtype` into a **flat rank-1** Tensor. *Migration:* `Fromfile` kept as a `DeprecationWarning` alias for one release. ### R.2a Python API (numpy-style)

### `save` / `load`

```
Tensor.save(fname)           -> None      # write structured binary  (was Save)
Tensor.load(fname)           -> Tensor    # static; read it back     (was Load)

Serialize a Tensor to / from a structured binary file (the `.cytn` format).

`save` writes this tensor to `fname`; `load` (a static constructor called as
`Tensor.load(path)`) reconstructs a value-equal tensor — same elements, shape,
dtype and contiguity (finding T-IO3). Provide the FULL path including the
extension: `load` opens the path verbatim and does not auto-append it (finding
T-IO4). Unlike UniTensor's loader (which corrupted a tensor NAME through a heap
over-read, UT-IO5), a dense Tensor stores no name, so this round-trip is
deterministically clean.

Parameters
----------
fname : str
    Destination (save) / source (load) file path, including the extension.

Returns
-------
None (save) / Tensor (load)

Notes
-----
Renamed from the Capitalized `Save`/`Load` (finding T-IO1); those spellings remain
as `DeprecationWarning` aliases for one minor release, then are removed. Through
cytnx 1.1.0 `save` auto-appended `.cytn` (a deprecated behavior) while `load` did
not, and the extension disagreed with UniTensor's `.cytnx` (finding T-IO4) — the
next version requires an explicit, unified extension. For raw headerless bytes
(no shape/dtype), see `tofile`/`fromfile`.

See Also
--------
tofile : write raw element bytes (headerless, numpy-fromfile-style).
```

### `tofile` / `fromfile`

```
Tensor.tofile(fname)                    -> None      # raw element bytes  (was Tofile)
Tensor.fromfile(fname, dtype, count=-1) -> Tensor    # static; flat rank-1 (was Fromfile)

Write / read RAW, HEADERLESS binary element data (numpy `tofile`/`fromfile`
parity).

`tofile` dumps this tensor's elements row-major with NO metadata (no magic, shape,
or dtype). `fromfile` reads `count` elements of the caller-supplied `dtype` back
into a FLAT rank-1 Tensor — it CANNOT recover the original shape or dtype (finding
T-IO5): a `(2, 3)` tensor written with `tofile` reloads as shape `[6]`, and you
must `reshape` it yourself. `dtype` is REQUIRED (the bytes carry no type tag);
`count=-1` (default) reads the whole file.

Parameters
----------
fname : str
    Destination (tofile) / source (fromfile) file path.
dtype : int
    (`fromfile` only) a cytnx.Type code for interpreting the raw bytes. Required.
count : int, optional
    (`fromfile` only) number of elements to read; -1 (default) reads all.

Returns
-------
None (tofile) / Tensor (fromfile — always rank-1)

Notes
-----
Renamed from the Capitalized `Tofile`/`Fromfile` (finding T-IO1); the old
spellings remain as `DeprecationWarning` aliases for one release. Use `save`/`load`
when shape and dtype fidelity matter; `tofile`/`fromfile` is the low-level raw path
for interop (e.g. with `numpy.fromfile`).

Pickle (`__getstate__`/`__setstate__`) is only half-implemented through 1.1.0
(finding T-IO2): the inherited `object.__getstate__` is present but `__setstate__`
is absent and `pickle.dumps(t)` raises `TypeError`. The next version either
implements BOTH hooks over the `.cytn` binary format or removes the lone
`__getstate__`; use `save`/`load` for persistence in the meantime.

See Also
--------
save : structured serialization that DOES preserve shape/dtype.
```

### R.2b C++ API (Doxygen)

The C++ `Save`/`Load`/`Tofile`/`Fromfile` already exist and bind directly; the
next version's changes are: (1) rename the members to `save`/`load`/`tofile`/
`fromfile` (T-IO1), (2) drop `Save`'s deprecated no-extension auto-append and
unify the extension with UniTensor (T-IO4), and (3) add pickle `__getstate__`/
`__setstate__` (Python side, over the same binary format) or drop the lone
`__getstate__` (T-IO2). The `.cytn` on-disk layout stores no name field, so there
is no `_Load` over-read to fix here (contrast UniTensor UT-IO5).

```cpp
/**
 * @brief Serialize this Tensor to a structured binary file (member `save`).
 * @details Writes magic + shape + contiguity + mapper + storage (Tensor.cpp
 *          _Save); load() reconstructs a value-equal tensor — same elements,
 *          shape, dtype and contiguity (finding T-IO3). Renamed from Save
 *          (finding T-IO1). Require the full path with a UNIFIED extension — no
 *          asymmetric `.cytn` auto-append (finding T-IO4). No name is stored, so
 *          there is no name over-read to fix here (contrast UniTensor UT-IO5).
 * @param fname destination path (including the extension).
 */
void save(const std::string &fname) const;

/**
 * @brief Deserialize a Tensor from a structured binary file (static `load`).
 * @details Reconstructs the tensor written by save(). Renamed from Load (finding
 *          T-IO1). Opens the path verbatim (no auto-append, finding T-IO4).
 * @param fname source path (including the extension).
 * @return the reconstructed Tensor.
 */
static Tensor load(const std::string &fname);

/**
 * @brief Write this Tensor's raw element bytes, headerless (member `tofile`).
 * @details Row-major element dump via Storage::Tofile — NO magic/shape/dtype
 *          (finding T-IO5). Renamed from Tofile (finding T-IO1). Use save() when
 *          shape/dtype must be preserved.
 * @param fname destination path.
 */
void tofile(const std::string &fname) const;

/**
 * @brief Read raw headerless bytes into a FLAT rank-1 Tensor (static `fromfile`).
 * @details Reads @p count elements of @p dtype via Storage::Fromfile. CANNOT
 *          recover the source shape or dtype — always returns a 1-D tensor; the
 *          caller must reshape and must supply the correct dtype (finding T-IO5).
 *          Renamed from Fromfile (finding T-IO1). Pickle __getstate__/__setstate__
 *          must be implemented fully over the save/load binary format or omitted
 *          entirely — never a lone inherited __getstate__ with no __setstate__
 *          (finding T-IO2).
 * @param fname source path.
 * @param dtype cytnx::Type code for interpreting the bytes (required).
 * @param count number of elements to read; -1 reads all.
 * @return a rank-1 Tensor of the requested dtype.
 */
static Tensor fromfile(const std::string &fname, const unsigned int &dtype,
                       const cytnx_int64 &count = -1);
```
